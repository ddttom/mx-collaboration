# Tools and Resources

Curated list of tools, frameworks, and resources for implementing Machine Experience (MX) patterns.

## Analysis and Audit Tools

### Web Audit Suite

**What:** Comprehensive website analysis implementing MX patterns

**Features:**

- robots.txt compliance checking
- Pa11y accessibility analysis
- LLM-based metrics
- Pattern extraction
- Regression detection

**Link:** [Web Audit Suite Repository](https://github.com/Digital-Domain-Technologies-Ltd/invisible-users/tree/main/packages/web-audit-suite)

**Use for:** Complete site audits, baseline assessment, tracking improvements

### Accessibility Validators

**Google Lighthouse:**

- Built into Chrome DevTools
- Accessibility, performance, SEO
- Free and widely used

**axe DevTools:**

- Browser extension
- Detailed accessibility reports
- Rule explanations

**WAVE:**

- Web accessibility evaluation tool
- Visual feedback
- Chrome/Firefox extensions

### Schema.org Validators

**Google Rich Results Test:**

- <https://search.google.com/test/rich-results>
- Validates structured data
- Shows how Google sees your markup

**Schema.org Validator:**

- <https://validator.schema.org/>
- Comprehensive Schema.org validation
- Checks syntax and required properties

**JSON-LD Playground:**

- <https://json-ld.org/playground/>
- Test and debug JSON-LD
- Visualize structured data

## Development Frameworks

### HTML/CSS

**Modern Normalize:**

- Consistent baseline across browsers
- Accessibility-focused resets

**Semantic HTML Resources:**

- MDN Web Docs: <https://developer.mozilla.org/en-US/docs/Web/HTML>
- HTML5 Doctor: <http://html5doctor.com/>

### JavaScript Frameworks

**React:**

- Component-based architecture
- Strong accessibility support with proper implementation

**Vue:**

- Progressive framework
- Good accessibility capabilities

**Svelte:**

- Compiled framework
- Clean semantic output

**Note:** Any framework can be MX-friendly if used with semantic HTML and accessibility in mind.

## Testing and Validation

### Screen Readers

**NVDA (Windows):**

- Free and open source
- Most popular on Windows
- <https://www.nvaccess.org/>

**JAWS (Windows):**

- Commercial screen reader
- Industry standard
- <https://www.freedomscientific.com/products/software/jaws/>

**VoiceOver (macOS/iOS):**

- Built into Apple devices
- Excellent for testing

**TalkBack (Android):**

- Built into Android
- Mobile testing

### Browser Testing

**BrowserStack:**

- Cross-browser testing
- Real devices
- Accessibility testing features

**Can I Use:**

- <https://caniuse.com/>
- Check browser support for features

## Standards and Specifications

### llms.txt

**Specification:** <https://llmstxt.org/>

**Purpose:** Standard file for providing context to LLMs

**Examples:** Appendix H in MX-Bible

### Universal Commerce Protocol (UCP)

**Website:** <https://ucp.dev>

**Purpose:** Standardized ecommerce APIs for AI agents

**Documentation:** Comprehensive protocol documentation

**Repository:** <https://github.com/Universal-Commerce-Protocol/ucp>

### WCAG 2.1

**Specification:** <https://www.w3.org/WAI/WCAG21/quickref/>

**Purpose:** Web accessibility guidelines

**Levels:** A, AA (target), AAA
